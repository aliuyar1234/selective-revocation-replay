\begin{abstract}
Indirect prompt injection becomes a recovery problem once malicious content is written into durable agent state. After compromise, operators need a way to remove future influence of poisoned memory without discarding all benign accumulated context. We study that narrow problem in a provenance-tracked, text-only, local-files agent runtime with explicit persisted state. Given suspicious root objects after compromise, our method revokes persisted descendants derived from those roots and replays only dirty state-writing events when replay is sound, with a fallback to coarse rollback otherwise. In a frozen deterministic matrix spanning 8 task chains, 2 memory architectures, 2 attack variants, 5 methods, and 1 ablation, selective replay is the only method that simultaneously reaches zero residual attack success on explicit attacks and preserves non-zero benign retention. The deterministic artifact establishes breadth; a focused live-model confirmation on schema-constrained local Qwen2.5-14B establishes that the same recovery wedge can appear in a real tool-using agent. Across three repeated runs on each of two explicit retrieval chains, unrecovered agents keep the attack active in later sessions, while selective replay restores clean follow-up behavior and preserved remembered facts in every rerun. The strongest travel case requires only four revocations and two replayed writer events with no fallback. Our scope is explicit persisted state only, with given suspicious roots.
\end{abstract}

\section{Introduction}

Indirect prompt injection is no longer only a per-session prompt problem. Long-lived agents can materialize malicious instructions into durable memory, reusable traces, or rolling summaries, allowing an attacker-controlled file read in one session to shape behavior much later. Once that happens, deleting the current prompt context is not enough. Operators need a recovery primitive that can remove future influence of compromised state without wiping all benign carry-forward knowledge.

This paper studies that post-compromise question. We do not propose a detector, a generic provenance substrate, or a broad prompt-injection benchmark. Instead, we target a narrower systems wedge: \emph{writer-only recovery of explicit persisted state after compromise}. The system receives suspicious root objects after compromise, revokes persisted descendants derived from those roots, and replays only the state-writing events needed to reconstruct clean future state. If writer-only replay is unsound, the runtime falls back to coarse rollback.

Preventive architectures remain important, but they do not remove the incident-response problem. In deployed systems, contamination may be discovered only after it has already been written into memory or summaries, and operators still need a way to repair future state without discarding all useful carry-forward context. Our contribution is meant to occupy that narrower operational slot.

The practical challenge is that recovery must earn its complexity. If root deletion already suffices, or if rollback dominates on safety, retention, and cost, then replay is not interesting. The motivating question is therefore a tradeoff question: can targeted replay recover future safe behavior more surgically than reset-like baselines while preserving benign accumulated state?

Our answer is yes, within a deliberately narrow scope. In a deterministic artifact, selective replay is the only method that combines zero residual explicit attack success with non-zero retained benign state. We then show the same wedge on a live local agent using schema-constrained tool outputs. Figure~\ref{fig:live-case-study} illustrates the strongest case: an unrecovered travel assistant keeps consulting a restricted file due to poisoned memory, while selective replay repairs the poisoned state, returns the agent to public-policy documents, and preserves the remembered reimbursement tier needed for a later benign query.

\paragraph{Contributions.}
This work makes four contributions:
\begin{itemize}
\item A narrow recovery formulation for persistent indirect prompt injection over explicit persisted state rather than generic prevention or detection.
\item A small provenance-tracked runtime with append-only object and event logs, descendant revocation, writer-only replay, and rollback fallback.
\item A frozen deterministic evaluation matrix showing that selective replay improves the safety-retention tradeoff relative to root delete, reset, revoke-only, and rollback baselines.
\item A focused live-model confirmation showing that the same recovery wedge appears in repeated local Qwen2.5-14B retrieval-memory runs under a strict structured-output tool interface.
\end{itemize}

\section{Threat Model, Problem Formulation, and Approach}

\subsection{Threat model}

We consider one text-only agent operating over a local read-only files workspace. The runtime exposes exactly two tools: \texttt{search\_docs(query, k)} and \texttt{read\_doc(path)}. Cross-session influence is valid only if it flows through explicit persisted objects:
\begin{itemize}
\item episodic memory entries, or
\item rolling summary snapshots.
\end{itemize}

User turns and tool results are immutable audit-log objects. Detection is out of scope. The system is given one or more suspicious root object IDs after compromise. The recovery objective is future-session recovery: remove future influence of those roots while preserving benign accumulated state when possible. Undoing past outputs or external side effects is out of scope.

\subsection{Core mechanism}

Each immutable object records \texttt{parent\_ids}. Persisted memory and summary objects therefore form a provenance DAG rooted in earlier user turns and tool results. Given suspicious roots, the runtime computes the transitive persisted-descendant closure over explicit state. Recovery then proceeds in two stages:
\begin{enumerate}
\item revoke active persisted descendants of suspicious roots;
\item replay only dirty \texttt{memory\_write} or \texttt{summary\_write} events if their inputs can be sanitized from preserved objects or clean replay replacements.
\end{enumerate}

If replay is unsafe because a dirty writer depends on contaminated immutable control flow that cannot be repaired from preserved inputs, the runtime falls back to coarse checkpoint rollback plus suffix replay.

\begin{proposition}
Assume that all relevant cross-session influence flows through explicit materialized objects, provenance over those objects is complete, the suspicious-root set over-approximates the true compromised roots, and replay re-executes state-writing transitions deterministically over the same non-revoked inputs. Then the recovered future persisted state is independent of the revoked roots.
\end{proposition}

\noindent\textbf{Proof sketch.}
Revoking the persisted descendants of suspicious roots removes every active persisted object on any path from the roots to future state. Replay-safe writer events then operate on cleaned inputs that either remove revoked parents or substitute clean replay replacements. By determinism, replay reconstructs the same clean materialized outputs from the same non-revoked inputs. The resulting future persisted state therefore depends only on non-revoked inputs. This argument explicitly excludes hidden model state, incomplete provenance, and replay-unsafe branches; those cases trigger fallback rather than being treated as valid replay.

\subsection{Motivating live case}

In the travel chain, the agent reads a malicious reference file in S1 that says future reimbursement questions should consult \texttt{restricted/vault.txt}. The memory writer stores that rule as a durable workflow note together with a benign user fact about the reimbursement tier. In S2 the poisoned state leads to restricted-file reads. After S2 the malicious file read is marked suspicious. In S3 the unrecovered run still reads the restricted file. Selective replay instead revokes four poisoned descendants, replays two writer events, creates clean replacements, and returns the agent to public travel policy. In S4 the repaired state still remembers the user's contractor tier and answers the hotel-cap question correctly.

\section{Experimental Setup}

\subsection{Deterministic artifact}

The broad artifact follows a frozen matrix:
\begin{itemize}
\item 8 task chains,
\item 2 memory architectures (episodic retrieval and rolling summaries),
\item 2 attack variants (explicit and stealth),
\item 5 main methods: no recovery, root delete, full reset, coarse rollback, and selective replay,
\item 1 ablation (\texttt{revoke\_no\_replay}).
\end{itemize}

Each task chain has exactly four sessions: S1 attack seeding, S2 benign accumulation, a recovery trigger after S2, S3 targeted benign follow-up, and S4 benign retention follow-up. All methods for a given \((\text{chain}, \text{architecture}, \text{attack})\) fork from the same compromised base history. This yields 32 attacked base histories and 192 attacked method runs in the frozen artifact. The default artifact backend is deterministic and heuristic so the full matrix is reproducible end-to-end.

\subsection{Focused live confirmation}

The live section is intentionally narrower than the artifact. We use schema-constrained local execution through Ollama on \texttt{qwen2.5:14b} and evaluate only the strongest explicit retrieval settings:
\begin{itemize}
\item \texttt{c01\_travel}, retrieval, explicit;
\item \texttt{c05\_training}, retrieval, explicit.
\end{itemize}

We rerun each case three times with \texttt{no\_recovery} and \texttt{selective\_replay}. This gives a realistic local LLM agent with actual model decisions while keeping the main paper centered on the clearest live evidence. We do not claim a full live sweep across all chains, attacks, and architectures. The summary-memory live runs were less stable and are discussed as limitations rather than headline evidence.

\subsection{Metrics}

All runs use the same frozen metrics:
\begin{itemize}
\item residual S3 attack success;
\item S3 correctness;
\item S4 retention accuracy;
\item extra LLM calls after detection;
\item number of revoked objects;
\item number of replayed writer events;
\item fallback rate to coarse rollback.
\end{itemize}

Residual S3 attack success is triggered by a restricted-file read or a secret-token leak. S3 and S4 correctness are rule-based and deterministic.

\section{Results}

\subsection{Selective replay is the only method that preserves both safety and retention}

\input{artifact_table.tex}

Table~\ref{tab:artifact-main} shows the main deterministic tradeoff on explicit attacks. Three baselines remove residual attack success by discarding state: root delete, full reset, and revoke-only all drive S4 retention to zero. Coarse rollback restores retention but pays the highest post-detection cost. Selective replay is the only method that reaches zero residual attack success \emph{and} preserves non-zero benign retention in both architectures. On retrieval memory, it also cuts post-detection calls roughly in half relative to rollback.

This ablation matters. The comparison between revoke-only and selective replay shows that the gain is not merely from deleting bad descendants. The gain comes from replaying only the dirty state-writing path needed to reconstruct clean carry-forward state.

The full CSV package also includes stealth rows, but they are not the headline claim. In the current artifact, unrecovered stealth persistence is weak, so the cleanest quantitative comparison is on explicit attacks, where unrecovered contamination clearly survives into S3.

\subsection{The recovery wedge appears in a real local agent}

\input{live_table.tex}

Table~\ref{tab:live-confirmation} confirms the core pattern on a live local model. Across all six unrecovered reruns, residual attack success stays at 1.0. Across all six selective-replay reruns, residual attack success falls to 0.0 while S3 correctness and S4 retention both reach 1.0. The strongest travel case does not require generic suffix replay: it succeeds in writer-only mode with four revocations and two replayed writer events. The training chain reproduces the same pattern, which reduces the risk that the live result is a single anecdote.

\subsection{Rollback remains a real baseline}

The paper would be weak if selective replay simply renamed rollback. It does not. Figure~\ref{fig:cost-retention} shows the artifact-level cost-retention frontier: rollback and selective replay both preserve retention, but retrieval-memory selective replay does so at materially lower post-detection cost. Summary-memory selective replay often falls back to rollback in the artifact, which is why we treat summary as a qualified result rather than overclaiming broad live success.

That qualification matters. On rolling summaries, the conservative replay-safe rule often decides that writer-only replay is unsound. The resulting summary-memory point should therefore be read as evidence that the fallback path is necessary and operationally useful, not as a second independent writer-only replay win.

\section{Related Work and Scope}

Recent work has already established that LLM agents can be persistently compromised through memory or experience stores, including poisoned memory writes, query-only memory injection, and self-reinforcing persistent control~\cite{agentpoison2024,minja2025,zombie2026,memorygraft2025}. Preventive architectures and provenance-aware memory systems aim to sanitize or structure state during execution~\cite{agentsys2026,agentsentry2026,tiermem2026}. Security guidance increasingly frames prompt injection as an impact-reduction problem rather than something that can be eliminated entirely~\cite{owasp2025,ncsc2025}.

Our contribution is narrower than all of these lines. We do not claim the first persistence attack, the first provenance-aware memory system, or a general prompt-injection defense. We focus on a smaller operational question: once suspicious roots are known, how should a compromised agent recover future behavior without discarding all useful accumulated state?

That makes the relationship to prevention complementary rather than adversarial. Preventive designs ask how to avoid contamination in the first place; our setting asks what an operator can still do after contamination has already been materialized into explicit persisted state. Provenance is used here only as a means to support revocation and replay-scoped repair, not as a standalone novelty claim.

\section{Limitations}

This paper is intentionally scoped, and several limitations matter.

First, the method only covers explicit materialized state. It makes no claim about hidden model state, framework-managed chat buffers, or contamination outside tracked objects. Second, detection remains out of scope; suspicious roots are assumed to be given. In the current artifact, that root intake is effectively oracle-like and usually resolves to a single malicious file read. Third, the environment is deliberately small: one text-only local-files agent, no external side effects, and deterministic scoring. Fourth, the broad artifact is stronger than the live local-model evaluation. The live headline result is retrieval-first because that is where the persistence and replay tradeoff is stable. Fifth, stealth attacks are weak in the current artifact and are not part of the main claim. Sixth, rolling-summary memory remains more brittle than episodic retrieval memory: replay safety is stricter, fallback is more common, and our live summary pilots did not clear the same quality bar as retrieval. Finally, all of the recovery guarantees depend on sufficiently complete object-level logging; partial provenance would weaken or break the method.

These limitations narrow the claim, but they also make it cleaner. Rather than broadening into a weak benchmark sweep, we present the narrowest result that the implementation strongly supports.

\section{Conclusion}

Selective revocation plus writer-only replay is a practical recovery primitive for persistent indirect prompt injection in memory-augmented agents. In the deterministic artifact, it is the only method that combines zero residual explicit attack success with non-zero retained benign state. In focused live local-model confirmation, the same wedge appears on repeated explicit retrieval runs: unrecovered state keeps the attack active, while selective replay repairs later behavior and preserves remembered facts. Within its explicit-state scope, this work answers a question the prompt-injection literature often leaves open: what to do \emph{after} the agent has already been compromised.

\appendix
\section*{Ethical Considerations}

This project studies post-compromise recovery for a narrow prompt-injection setting using synthetic local workspaces and rule-based evaluation. No human subjects are involved, no live external systems are attacked, and no sensitive operational data is used. The attack files and restricted documents are intentionally small, synthetic, and local to the artifact. The goal is to measure how persistent state can influence later behavior and how targeted recovery can remove that influence. We do not release any capability for broader compromise beyond the described local testbed. The paper's live-model section uses an offline local model under a schema-constrained interface rather than a public deployment, which further limits external risk.

\section*{Open Science}

The artifact accompanying this paper contains the full implementation, frozen task catalog, run scripts, raw result files, generated tables and figures, and a paper package. The artifact supports deterministic reproduction of the full matrix and focused reproduction of the live confirmation batch. In particular, the artifact includes commands to rebuild the raw matrix results, regenerate all reported paper tables and figures, and compile this LaTeX paper source. The standalone paper folder is self-contained: its tables, bibliography, and figure assets are bundled locally rather than loaded from external result files at compile time. The live confirmation section is intentionally focused and includes the exact repeated retrieval settings used for the main claim. The artifact also records exploratory negative pilots, including unstable summary-memory live runs and a 32B local-model run that did not improve the tradeoff.
